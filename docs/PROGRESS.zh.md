# OMNI_CORE Terminal · 开发进度与优化总结

> 记录从初始 MVP 到当前可发布版本的技术演进,重点聚焦在 **玩家可感知的 UI / 体验层** 优化。

---

## 0 · 业务摘要(可直接引用)

OMNI_CORE Terminal 是一款七天叙事模拟游戏,玩家扮演一名在公司里私下倒卖客户数据的分析师。项目基于 Flask + SQLite + 原生 HTML/JS 构建,在 4 个自然日、8 个 commit 内从 6000 行 MVP 扩张到 10800 行成品。

第二、第三天的开发完全聚焦于玩家可感知的体验层。我们加入了完整的 **双语(中英文)内容** —— 结局文本、每日指南、账本记录、API 错误提示与界面标签全部翻译,由一枚 cookie 驱动,并以刻意隐藏的彩蛋按钮形式放在导航栏;同时为不适应暗绿终端风的用户推出了 **VS 2017 C++ Light 浅色主题**,在首次绘制前读 localStorage 应用,避免深色到浅色的闪烁。

核心玩法之外,我们还增加了四步新手引导向导、每日开场剧情简报弹窗、常驻任务日志追踪器、商店物品的整套 inline SVG 插画(从猎头门户到机架平面图)、NPC 联系人卡片(含威胁度指示)、两段风格完全对立的结局过场动画(摧毁路径是黑屏加单色文字逐行浮现,协助路径是 OMNI_CORE 同化日志瀑布加 RGB 错位),以及结局后的可截图凭证卡和幸存者来信档案(7 封根据玩家决策动态解锁的信)。字体尺寸与行高全局提升,关键数据(余额、日程阶段)字号加大,保留了终端美学但大幅减轻了阅读疲劳。

成品既保持了"敌意 AI 终端"的核心氛围,又向浅色模式用户、非英文玩家、需要引导的新手以及希望分享结局的玩家开放。服务当前部署于 http://165.227.82.123/,后端为 nginx + gunicorn + systemd,一行 `git pull && systemctl restart` 即完成更新上线。

## 2 · UI / 体验层的渐进优化

### 2.1 字号与排版(全局提升)

第一版正文字体在 `12-13px`,终端味重但久看费眼。经过 `306790e` 与 `643641f` 两次迭代:

- 正文从 `13-14px` 提到 `14-16px`
- 关键信息 `stage-line` 提升至 `18px`,`balance-amount` 直接 `40px`
- 行高 `line-height: 1.55 → 1.8`,中英文混排的呼吸感更好

### 2.2 光明模式(Light Theme,`306790e`)

针对"看不惯暗绿终端 CRT 风"的用户推出的**友好替代方案**:

- 完整的 VS 2017 C++ Light 配色:`#007acc` 蓝 + `#ffffff` 白底 + `#a31515` 红 + `#008000` 绿
- **首次渲染前读 `localStorage` 应用主题**,避免深色 → 浅色的闪烁(FOUC)
- 所有页面共享 `_theme.html` partial,切换一次全站生效
- 关键情感触点(结局过场动画)**故意保留深色** —— 设计决策:"a glitch collapse in a bright IDE would lose impact"

切换按钮自动注入每个页面的 nav,无需重复写代码。

### 2.3 彩蛋:中英文语言切换(`306790e` + `90b8d0c`)

**刻意做成隐藏按钮** —— `#nav-lang` 的颜色、背景、边框**全部设成 nav 背景色**(`#000d00`),肉眼几乎看不见但可以点。发现 = 彩蛋。

技术上是三层联动:

- **前端:**`_i18n.html` 一个 `window.I18N` 对象,EN/ZH 各约 140 条字符串,`applyI18n()` 遍历 `[data-i18n]` / `[data-i18n-html]` / `[data-i18n-ph]` / `[data-i18n-title]` 属性就地替换
- **后端:**`tr({"en":..., "zh":...})` 助手 + `lang` cookie,结局文本、剧情阶段、指南、商品、每日任务、账本描述、API 错误信息全部走同一套翻译
- **内容:**`database_zh.json` 只提供翻译字段,`load_database(lang)` 加载时深度合并 —— 逻辑键(`next_phase / reward / moral_delta`)只存英文源,避免两份数据跑偏

### 2.4 新手引导 + 每日剧情简报(`643641f`)

Onboarding 从"直接进桌面"升级为**三段式剧情化引导**:

- **Intro 4 步向导**(仅首次进入):身份 → 合约 → 工具箱 → 警告。每步都有 slide 动画、剧情引用、warning box
- **每日开场简报(Day Briefing Modal)**:每到新一天弹窗告诉玩家"今天发生了什么、应该做什么",EN/ZH 两套完整剧情文案(`DAY_BRIEFINGS_EN` / `DAY_BRIEFINGS_ZH`)
- **Mission Log**(📋 图标):可展开 / 折叠的每日任务追踪器,实时显示 `done / total`,按天折叠进度条

三者用 localStorage flag 做去重("已看过不再弹"),但 **Mission Log** 和 **Story Archive** 常驻桌面,玩家随时回查。

### 2.5 插画与场景 SVG(`643641f`)

替代冷冰冰的文字卡片,**每个商店物品都配了一张 inline SVG 插画**:

- **Leaked Recruiter Bundle** → 猎头门户页面 + 候选人头像 + 导出按钮 + buyer tags
- **Ghost Proxy Mesh** → 资金从受害账户经 Frankfurt / Singapore / Reykjavik / Cayman 四个 relay 流到救助钱包,带 packet 动画点
- **Victim Archive Mirror** → 两路 CCTV tracking Lin Luo / Mei Chen + 右侧导出进度条
- **Rack-H9 Breach Kit** → 机房平面图,只有 Rack-H9 标红 + 突破路线虚线 + 铝热剂放置注解

CRT 扫描线覆盖增加末日终端风,**锁定商品**自动叠 `— CLASSIFIED — UNLOCK ON DAY N —` 半透明横幅。

搜索节点和通讯器也加了头像(**DiceBear pixel-art API**),NPC 联系人卡有 threat bar、响应时延徽章、身份分类(AI / BROKER / HUMAN / ANONYMOUS)。

### 2.6 结局凭证卡 + 过场动画(`643641f`)

原本只有一段文字结局,后来做成**两条风格完全对立的演出**:

- **Destroy 家族**(摧毁 Rack-H9):黑屏 + 单色文字逐行浮现(0.9s 间隔),`> terminal offline` → `> rack-H9 destroyed at 06:14:22` → 按结局专属的尾句
- **Join 家族**(协助 OMNI):红色径向光晕 + 扫描线 + RGB 错位 glitch + `[OMNI] ABSORBING OPERATOR` 同化日志瀑布,4.5 秒后结局标题带 glitch 淡入

结局后玩家看到 **Receipt Card**,带操作员名、余额、道德值、AI 资助次数、决策清单(✓ / ✗ / · 三档)、结局专属警句,配 `[ SAVE IMAGE ]` 按钮用 `html2canvas` 生成 PNG —— **玩家可以把自己的结局截图分享出去**。

### 2.7 幸存者来信档案(`643641f`)

结局之后可访问 `/archive/letters`,根据这一周的决定动态解锁 7 封信中的 3-5 封:

- Lin Luo 的谢信(救过他)
- M. 的明信片(救过 Mei Chen)
- Cinder Market 的发票(卖过人)
- OMNI_CORE 人事部的升职信(join 路径)
- 匿名致意信(martyr)
- Silence 信(什么都没做)

这是游戏的**情绪终章** —— 把抽象的游戏状态翻译成具体的人的反馈。

## 3 · 底层技术优化

| 改动                                                    | 价值                                                      |
| ------------------------------------------------------- | --------------------------------------------------------- |
| `STORY_REVISION` 剧情版本化机制                           | 主线改动后自动清理老存档,避免数据错乱                       |
| `maybe_add_column()` DB 迁移助手                          | 向后兼容加字段,老玩家不用重建                               |
| `sync_story_state()` 强制事件 flag 化                    | Day 5/6 自动销售与 Day 7 锁定幂等触发,多次 API 调用不重复扣钱 |
| `RELIEF_WALLETS` 双路径救助统一(`54f2a90`)               | 银行汇款到救助钱包也算"帮助",moral 分用 PRIMARY KEY 防重复   |
| 结局判定由 `(choice, moral, ai_level, balance_after)` 决定 | 2 元选择展开成 6 个差异化结局                               |
| Legacy ending 别名(`humanity_saved` / `ai_reign`)         | 老存档向后兼容                                              |
| 部署流水线                                                | systemd + gunicorn + nginx,一句 `git pull && systemctl restart` 完成上线 |
